# django-cotton design rules

1. prefer HTML syntax `<c-card arg="Value">...</c-card>` over django native template tags `{% cotton card arg="Value" %}`.
2. Utilize `<c-vars>` heavily to create well-documented and complete components
3. Use compound components when appropriate to ensure concise components with proper design separation.
